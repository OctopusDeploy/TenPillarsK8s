Epilogue

One of the primary motivations for writing this book was to identify gaps in the functionality provided by Octopus. We are always looking for ways to improve the product, and if you have reached this point I imagine you have found a few use cases that Octopus just doesn't quite support.

We're always interested to hear your feedback. If you have provided feedback in the past that was not actioned, please accept my apology. The volume of feature requests we receive often outweighs our ability to implement them. As Octopus grows though we find ourselves in a position to build more features and improve rough edges, and so even suggestions made some time ago are being revisited. We expect to be able to deliver some incredible new functionality in Octopus over the next few years, so now is a good time to be involved with the conversation.

The links below provide ways you can get in touch with us:

  • Community Slack - https://octopususergroup.slack.com/
  • Support - https://octopus.com/support
  • User Voice - https://octopusdeploy.uservoice.com/

The source code to this book can be found at https://github.com/OctopusDeploy/TenPillarsK8s.

You can reach me directly via Twitter at @mjcasperson.

Thank you for taking the time to read this book. Like all good software projects I expect this book will continue to evolve over time. If it is possible for a book to feature creep, I'll probably find a way to do that too :). This book is being built with continuous delivery, with each new version made publicly available at https://github.com/OctopusDeploy/TenPillarsK8s/releases, so keep an eye out for updates.

Happy deployments!
